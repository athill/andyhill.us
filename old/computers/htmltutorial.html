This tutorial is going to cover the basics of XHTML, HTML is the Hyper Text Markup Language and XHTML is HTML that conforms to the rules of XML. XHTML is used to format web pages. It uses tags surrounding text elements to indicate how to format the text. For example, to add write a word in italics, you would enclose the word in tags like this:
<em>...</em>
All XHTML markup tags are enclosed in less-than (<) and greater-than(>) signs. For convenience, these are often referred to as 'angle brackets' to differentiate them from parentheses(()), square brackets([]), and curly braces({}). Angle brackets were probably chosen for this task as they are less common than the other delimiters and allow you to use the other delimiters without worrying about it. All angle brackets in an HTML document are considered delimiters, so if you actually want to display a less-than or greater-than sign, you must use an escape sequence(for example, &lt;, for less-than). We'll discuss that more later.

For now, just remember that all HTML tags are enclosed in angle brackets. Most HTML tags come in pairs, like the italics tag above. In this case, <em> means "start doing italics here" and </em>(with a '/' following the '<') means "stop doing italics here". Another simple tag that uses this formula is:
  • <strong>bold</strong>
Tags that don't come in pairs are called "empty tags." Empty tags must close themselves by having a " /" before the closing angle bracket for example a horizontal rule (a line across the page) is expressed as:
  • <hr />

Document Structure

 In addition to formatting text, tags are also used to format the document itself. Each HTML document starts with <html> and ends with </html>. Within the document, there is a head and a body, which are delimited by <head>...</head> and <body>...</body>, respectively. Within the head, is the title, scripts(discussed later), and meta tags(also discussed later). The title is the text you would like to appear in the browser's title bar(in the case of this page, "Andy's XHTML Tutorial"), enclosed in matching <title>...</title> tags. The body holds the substance of the HTML document. So a skeletal HTML document would appear as such:

<html>
<head>
<title>Your title here</title>
</head>
<body>
Substance of HTML document
</body>
</html>

A couple of worthy notes:
  • XHTML is case sensitive. <em> is not the same as <EM>. You must use lower case for all tags
  • Tags must be properly nested. For example, if you have <stong> followed by <em> followed by text, you must first close with </em>, followed by </strong>. You must close inner tags before outer tags.

Formatting

The Font Tag

 The font tag is deprecated in XHTML. It is recommended you use Cascading Style Sheets(CSS) to achieve the formatting discussed in this section. However, the color section is still relevant. In addition to italics, underscore, and bold type, HTML lets you decide the size and color of the text. This is done using the font tag. For example, <font size="4" color="blue">...</font> would create text that looks like this. Size and color are called attributes of the font tag. The font tag can take a color attribute, a size attribute, or both.

The size attribute can take a number of forms:
  • size="n" where n is a number between 1 and 7 sets the size of the font to that size.
  • size="+n" or size=-n increases or decreases the current font size by n.
  • size="word" where word is big, small, large, or tiny, sets the font appropriately to the word.
The color attribute can take 2 forms:
  • color="word" where word can be black, blue, cyan, gray, green, lime, magenta, maroon, navy, olive, orange, purple, red, teal, silver, white, or yellow.
  • color="code" where code is a pound sign(#) followed by the six character hexadecimal equivelent of the desired color.
The hexa-what? you may ask. What this means is an encoding of the RGB(Red-Green-Blue) value of the color. Hexadecimal differs from decimal in that rather than going from 0-9 to 10, hexadecimal goes from 0-15 and then to 10. To make this less confusing(maybe), after 9, hexadecimal numbering actually goes to a-f, where f=15. So counting to decimal 16 in hexadecimal goes: 1,2,3,4,5,6,7,8,9,a,b,c,d,e,f,10. 16 in decimal is equal to 10 in hexadecimal.

So having a 6 character hexadecimal code for a color allows you to choose between 256 values each for the red, green, and blue components of the color, the higher the value, the greater intensity of the color. Some examples might help:
  • #000000 is black.
  • #FFFFFF is white.
  • #FF0000 is red.
  • #FFFF00 is yellow.
Any of these colors could just as easily be called by name, but more original colors, such as forest green(#006500) require hexadecimal annotation. Back to Top

Headers

 HTML also provides 7 levels of headers of the form <hn>...</hn> where n is a number between 1 and 7. Headers differ from using the font tag with the size attribute in that
  • The text appears on its own line.
  • Headers may take an align attribute of left, right, or center.
  • As opposed to font size, h1 is actually the largest, h7, the smallest.
For example, <h3 align="center">My Header</h3> displays:

The default alignment for headers is left.

The Body Tag

 You can also format an entire HTML document using the body tag. Its attributes include bgcolor to set the background color for the page and text to set the text color for the page. link, alink, and vlink can be used to set the color of links, active links, and visited links respectively(links are discussed soon). All these attributes use the same color representations as the font color attribute. So, for example, <body bgcolor="#000000" text="#FFFFFF" link="#FFFF00">...</body> makes the background color of the page black, the default text is white, and all links which have not been clicked are yellow.

Line Breaks and White Space

 HTML ignores line breaks and all white space after the first in a sequence. By default, HTML simply wraps text according to the width of your browser window. If you want to insert a line break, there are two common ways to do it:
  • <br /> inserts a simple line break.
  • <p>(paragraph)</p> inserts empty lines above and below the text.
Note that unlike most XHTML tags, there is no closing tag for the <br /> tag.

White space is more difficult to manufacture in HTML. The most common way to add white space is the escape sequence, &nbsp;(no-break white space). Other options include using tables and images(discussed later).

Links and Anchors

 One of the greatest things about HTML is the ability to jump from page to page or within pages by clicking on links. Links are of the form <a href="address">text to click on</a> where address can be a relative or absolute url(Uniform Resource Locator).

Relative urls are urls that are contained on your site. So for example, if I wanted to link to a page in the same directory as the one you're currently reading called page2.html, I could just write <a href="page2.html">page2</a>. If the file is the directory above the page this one, the convention '../' means move up one directory (i.e., if page2.html is in the directory above the page you're currently reading, you could link to it as such: <a href="../page2.html">page2</a>). If the document is below the current document, say in a folder called folder2, You would link to it as <a href="folder2/page2.html">page2</a>

Absolute urls are links to pages not on your site. For these you need the full address of the intended page, starting with 'http://'. For example, to link to the Indiana University Bloomington Homepage, you would type <a href="http://www.indiana.edu">homepage</a>. Similar rules apply as to relative links for going inside folders, but you usually cannot go up(../), as in relative urls.
You may also use the target attribute to decide which window the link will open in. target="_blank" opens in a new window, _parent, _self, and _top are also used, but more often in framesets.

Anchors are links within a page. A location on a page can be marked usng the syntax <a name=name>name</a> This location can be linked to using the syntax <a href="#name">name</a>. If name is on another page, say page2.html, use the syntax <a href="page2.html#name">name</a>.

Images

 Linking to images is another important part of HTML. The syntax is simply, <img src="image-file" />. You can also use an align attribute, the options being left, right, and center(default is left). In addition, it is recommended to use height and width attributes. These are given in pixels and can indicate the actual dimensions of the image or can be used to shrink or expand the image.

Lists

 Lists can be ordered or unordered. To begin an unordered list, use <ul>. Beginning an ordered list begins with a <ol> tag. Either type of list needs a complementary closing tag. Either type of list may take a type attribute to indicate the symbol to be used.
  • Unordered lists may take disc, circle, or square types. This is also the default sequence of symbols for nested ordered lists.
  • Ordered lists can take type A, a, I, i, or 1, which indicate the starting value of the list. The default is 1.
Each item in the list is started with an <li> tag, which does not need to be closedi for most browsers, but does to be XHTML compliant.
A list of definitions can also be created between <dl>...</dl> tags. In this case, <dt> is used for the term to be defined, and <dd> for the definition itself. Neither of these tags needs to be closed.

Tables

 Tables are a powerful way to control the layout of XHTML pages. The basic layout is:

<table>
  <tr>
    <td>
    </td>
    ...
  </tr>
  ...
</table>


where tr is a table row and td is a table column. A table can have any number of rows and columns, but the number of rows columns must be consistent throughout the table.
The table

  <tr>
    <td>
	One
    </td>
    <td>
	Two
    </td>
  </tr>
  <tr>
    <td>
	Three
    </td>
    <td>
	Four
    </td>
  </tr>
</table>
  shows up as
one two
threefour
This obstacle can be avoided by using colspan the rowspan attributes of the td tag.
The table

  <tr>
    <td>
	One
    </td>
    <td>
	Two
    </td>
  </tr>
  <tr>
    <td colspan="2">
	Three
    </td>
  </tr>
</table>
  shows up as
one two
three
The table tag can take several attributes. Border=n creates a border of n pixels. Border color can be set for the entire border as bordercolor=color where color can be the same values used in the font tag. You can also use the align attribute with the values left, right, or center. The width of the table can also be set. cellspacing(space between cells) and cellpadding(space around the contents of cells) can be indicated in pixels. Finally, you can choose which sides of the frame to display using the frame attribute which can take the following values: void(no borders), above, below, hsides, vsides, rhs, lhs, or box(default) and rules, for determining which inner borders to show. Possible values are none, rows, cols, and all(default).

The td tag also has several attributes. These include height, width, align and valign(for the cell's contents), bgcolor, and background(which takes an image).

Tables can also be nested.

Frames

 Frames allow you to have more than one page on your screen at a time. This can be useful if you want to have a static menu that navigates the rest of the page. The frame declaration goes between the closing head and opening body tags. A frameset starts with a <frameset> tag, which takes either a rows or cols attribute, depending on whether you want your frames to appear as rows or columns. The value of the rows or cols attibute is a quoted list containing either the length or width in pixels of each frame or the percentage of screen width or height for the frame to occupy. One of the elements in the quoted list can be an asterisk(*), which indicates that that frame take up any space not occupied by the other frames. Following the opening frameset tag are a series of frame tags, one for each element in the list in the frame tag. Frame tags have two attributes, a name field used to identify the frame and an src field which indicates the url of the frame. Finally, a </frameset> tag closes the frameset.

For example:
<frameset cols="100, *">   <frame name="menu" src="menu.html" />
  <frame name="content" src="http://www.mypage.com/pageone.html /">
</frameset>

Creates a page with a 100 pixel wide menu and content filling the rest of the screen. Cols go left to right, rows go top to bottom. In order to make the links on the menu change the content in the comment frame, use the target attibute of the a href tag(i.e., <a href="http://www.mypage.com/page2" target="content">). To jump out of a frameset, use the predefined "_top" value for target. You can also set target to _self, to open in the current frame, or _parent in a nested frameset. If you're worried about browsers that don't support frames(which are increasingly rare), you may supply alternate content in the body between <noframes> and </noframes> tags. Framesets can also be nested to allow both rows and columns of frames. The frameset tag has a couple other attributes, namely, bordercolor, which takes either predefined colors or hexadecimal color definitions and border, which indicates the width, in pixels, of the borders between frames(5 is default, 0 removes borders completely). The frame tag also has other attributes. Each frame has a border of 8 pixels on each side. This default can be changed using the marginwidth and marginheight attributes. Whether a user can scroll on a frame is usually decided by the frame's content compared to the size of the frame. However, this can be altered using the scrolling attribute, which can take a yes or no value. Finally, adding the attribute "noresize"(with no value) can prevent users from resizing frames, which they could normally do.

Forms

 Forms are used to transfer user data. Forms generally have an action attribute that indicates a script(such as Perl, PHP, ASP, or JSP) that will process the form. If you're just starting with forms and don't know any scripting language, you can also use action="mailto:your@email.address" to send the submitted data to your email account. Forms are enclosed in corresponding <form> tags and can include the following, in addition to other valid HTML elements:
  • Input elements. These are standalone attributes that require a parameter called type. The type can be text for standard text; password, for text that shows up as asterisks on the user's screen; and file, which adds a 'browse' button, which users may click to find a file, or they may specify the path to a file in the text box. Either way, the file is uploaded to the server. Other attributes include 'name', which specifies the name the value is associated with, and value, for specifying a default value. for example:

      </form>
