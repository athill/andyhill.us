Technologies used on this site:

  • XHTML - Hyper-text Mark-up Language that conforms to XML standards. Basically, this is code that tells the browser the logical structure of the document. For example, what you are now reading is an item in an unordered list, which is inside a paragraph, which is in the body of the document.
  • CSS - Cascading Style Sheets. Rules in the style sheet give presentational information to the page. For example, the background is black, the text is white, and th elinks are yellow, but turn blue when you roll your mouse pointer over them.
  • JavaScript - Technically, this should probably be called ECMAScript (don't ask). Anyway, this is a client-side scripting language, meaning it is interpreted by the web browser. JavaScript is responsible for about anything that changes on a web page that doesn't require the page to reload (outside of embedded stuff like Flash and JavaApplets). Examples are roll-over images, the clock at the top of this page, form validation popups.
  • PHP - PHP Hyper-text Processor. PHP is a server-side scripting language. This code is interpreted on the server and generates the content the sent to the browser.
  • Perl - Practical Extraction and Report Language. Perl is another server-side scripting language.
  • MySQL - SQL stands for Stuctured Query Language. MySQL is a database, meaning a set of tables where you define the names and types of the columns. You use SQL both to create the tables and to add and retrieve data from the tables. For example, in my blog, each entry is a row in the a table that has columns for the date, the content, etc. I then use Perl to retrieve the entries and format them for display.
  • Java Applets - Java is an application programming language. Java Applets are mini-applications that can be embedded on a web page.
  • Flash - Flash is an application made by Adobe that allows you to create sophisticated content that can be embedded on a web page, such as animation and games.
